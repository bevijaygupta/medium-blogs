### The Building Blocks of Networking {#2538 .graf .graf--h3 .graf--leading name="2538"}

### 1. Server {#9550 .graf .graf--h3 .graf-after--h3 name="9550"}

A **server** is a powerful computer or system that provides data,
services, or resources to other devices, known as **clients**, on a
network. Servers play a pivotal role in managing web traffic, hosting
applications, and storing data.

#### Types of Servers: {#c62d .graf .graf--h4 .graf-after--p name="c62d"}

- [**Web Servers:** Host websites and deliver web pages to
  users.]{#c5b0}
- [**Database Servers:** Manage and provide access to structured
  data.]{#6b82}
- [**Mail Servers:** Handle email delivery and retrieval.]{#264e}
- [**File Servers:** Store and manage files accessible over a
  network.]{#b080}

#### Example in Use: {#0007 .graf .graf--h4 .graf-after--li name="0007"}

When you access Google, your browser sends a request to Google's web
server, which processes the request and sends back the search results.

### 2. User {#623b .graf .graf--h3 .graf--leading name="623b"}

A **user** is any individual interacting with a computer or network
service to access resources. Users are the end recipients of data
delivered by servers.

#### Roles of Users: {#223c .graf .graf--h4 .graf-after--p name="223c"}

- [Initiate network requests (e.g., browsing a website).]{#5ba0}
- [Consume services (e.g., streaming video, downloading files).]{#da4e}
- [Input data into applications (e.g., filling out forms).]{#3d1c}

### 3. Sender {#95e4 .graf .graf--h3 .graf--leading name="95e4"}

The **sender** is the entity (device or individual) that initiates the
transmission of data across a network. Senders play a crucial role in
starting communication processes.

#### Use Case: {#b6ea .graf .graf--h4 .graf-after--p name="b6ea"}

When an email is sent, the device initiating the process acts as the
sender, transmitting the message to the recipient through an SMTP
server.

### Core Network Protocols: Enabling Communication {#2740 .graf .graf--h3 .graf--leading name="2740"}

### 4. HTTP (Hypertext Transfer Protocol) {#9d94 .graf .graf--h3 .graf-after--h3 name="9d94"}

**HTTP** is the foundation of data communication on the internet,
facilitating the transfer of resources like HTML documents, images, and
videos. It operates on a request-response model, where the client
(browser) sends a request to the server, and the server responds with
the requested data.

#### Example Workflow: {#78cf .graf .graf--h4 .graf-after--p name="78cf"}

- [The browser sends a GET request for a webpage.]{#3ac2}
- [The server responds with an HTTP response containing the requested
  page.]{#ba76}

### 5. HTTPS (Hypertext Transfer Protocol Secure) {#0d78 .graf .graf--h3 .graf--leading name="0d78"}

**HTTPS** is a secure version of HTTP that encrypts data during
transmission using protocols like TLS (Transport Layer Security). This
encryption ensures that sensitive data, such as login credentials and
financial information, remains private.

#### Importance: {#2c5b .graf .graf--h4 .graf-after--p name="2c5b"}

- [Protects against eavesdropping and man-in-the-middle attacks.]{#8dd0}
- [Ensures the authenticity of websites through digital
  certificates.]{#9ea9}

#### Example in Use: {#c28a .graf .graf--h4 .graf-after--li name="c28a"}

Online banking portals use HTTPS to safeguard user credentials during
login.

### 6. WebSocket {#8c2d .graf .graf--h3 .graf--leading name="8c2d"}

**WebSocket** enables real-time, two-way communication between a client
and a server over a single TCP connection. Unlike HTTP, which requires a
request for every response, WebSocket maintains a continuous connection,
making it ideal for dynamic applications.

#### Applications: {#f1a8 .graf .graf--h4 .graf-after--p name="f1a8"}

- [Online gaming]{#2a12}
- [Chat applications (e.g., Slack, WhatsApp Web)]{#673a}
- [Live stock market updates]{#fc6e}

### 7. FTP (File Transfer Protocol) {#fc8b .graf .graf--h3 .graf--leading name="fc8b"}

**FTP** is a protocol designed for transferring files between computers
on a network. It allows users to upload, download, and manage files on
remote servers.

#### Key Features: {#f5a5 .graf .graf--h4 .graf-after--p name="f5a5"}

- [Supports user authentication.]{#a509}
- [Enables bulk file transfers.]{#a3c0}

#### Example: {#3614 .graf .graf--h4 .graf-after--li name="3614"}

Web developers use FTP clients like FileZilla to upload website files to
hosting servers.

### 8. HTTP/3 (QUIC) {#fffa .graf .graf--h3 .graf--leading name="fffa"}

The latest iteration of HTTP, **HTTP/3**, leverages the QUIC protocol to
provide faster and more reliable connections. Unlike its predecessor,
HTTP/3 eliminates the need for TCP, using UDP for low-latency
communication.

#### Advantages: {#4b64 .graf .graf--h4 .graf-after--p name="4b64"}

- [Faster page loads]{#7944}
- [Improved performance in mobile networks]{#2795}
- [Reduced connection latency]{#257c}

#### Use Case: {#d3e1 .graf .graf--h4 .graf-after--li name="d3e1"}

Modern websites like YouTube and Google employ HTTP/3 for enhanced user
experiences.

### 9. TCP (Transmission Control Protocol) {#a557 .graf .graf--h3 .graf--leading name="a557"}

**TCP** ensures reliable, error-checked data transmission between
devices. It establishes a connection, segments data into packets, and
reassembles them at the destination.

#### Features: {#402a .graf .graf--h4 .graf-after--p name="402a"}

- [Guaranteed delivery]{#3502}
- [Error detection and correction]{#b550}
- [Sequencing of data packets]{#be45}

#### Applications: {#9bd0 .graf .graf--h4 .graf-after--li name="9bd0"}

TCP is used in applications where data integrity is critical, such as
file transfers and emails.

### 10. UDP (User Datagram Protocol) {#79a2 .graf .graf--h3 .graf--leading name="79a2"}

**UDP** offers faster data transmission by foregoing error-checking and
guaranteed delivery, making it suitable for applications where speed is
prioritized over reliability.

#### Applications: {#52a0 .graf .graf--h4 .graf-after--p name="52a0"}

- [Video streaming]{#c323}
- [Online gaming]{#457c}
- [Voice over IP (VoIP)]{#f25f}

#### Example: {#c7e6 .graf .graf--h4 .graf-after--li name="c7e6"}

Netflix uses UDP for faster video buffering, ensuring smooth streaming
even in fluctuating network conditions.

### 11. SMTP (Simple Mail Transfer Protocol) {#256e .graf .graf--h3 .graf--leading name="256e"}

**SMTP** is the protocol for sending emails between mail servers. It
works alongside other protocols, such as **IMAP** and **POP3**, for
complete email functionality.

#### Example Workflow: {#a5d3 .graf .graf--h4 .graf-after--p name="a5d3"}

- [The sender's email client sends the message to the SMTP
  server.]{#c714}
- [The SMTP server forwards the email to the recipient's mail
  server.]{#83b2}

### Key Networking Concepts: Ensuring Secure and Reliable Communication {#274c .graf .graf--h3 .graf--leading name="274c"}

### 12. TCP Connection {#5b47 .graf .graf--h3 .graf-after--h3 name="5b47"}

A **TCP connection** is a virtual connection established between two
devices using TCP, ensuring reliable communication. The **three-way
handshake** process (SYN, SYN-ACK, ACK) initiates this connection.

#### Use Case: {#4e93 .graf .graf--h4 .graf-after--p name="4e93"}

When you log in to an online banking platform, a TCP connection ensures
that your credentials are securely transmitted.

### 13. HTTP Response {#39c0 .graf .graf--h3 .graf--leading name="39c0"}

An **HTTP response** is the data sent by a server after processing an
HTTP request. It includes:

- [**Status codes:** Indicate the outcome (e.g., 200 OK, 404 Not
  Found).]{#f58c}
- [**Headers:** Contain metadata.]{#4e9e}
- [**Body:** Contains the requested content (e.g., HTML, JSON).]{#efd5}

#### Example: {#6cc1 .graf .graf--h4 .graf-after--li name="6cc1"}

A search query on Google returns an HTTP response with the requested
search results.

### 14. Public Key {#738d .graf .graf--h3 .graf--leading name="738d"}

A **public key** is a cryptographic key used in public-key encryption
systems to facilitate secure communication. Paired with a private key,
it enables encrypted data exchange.

#### Applications: {#b4bc .graf .graf--h4 .graf-after--p name="b4bc"}

- [**TLS/SSL Certificates:** Secure HTTPS connections.]{#c827}
- [**PGP Encryption:** Secure email communication.]{#06a8}

#### Use Case: {#1678 .graf .graf--h4 .graf-after--li name="1678"}

When you connect to a secure website, the server's public key encrypts
data transmitted to it.

### Practical Applications of Networking Terms {#38f1 .graf .graf--h3 .graf--leading name="38f1"}

### Web Development {#745f .graf .graf--h3 .graf-after--h3 name="745f"}

Understanding protocols like **HTTP**, **HTTPS**, and **WebSocket** is
essential for building responsive and secure web applications.

### Network Administration {#84dc .graf .graf--h3 .graf-after--p name="84dc"}

Admins rely on protocols like **TCP**, **UDP**, and **FTP** to ensure
smooth data flow and system operations.

### Cybersecurity {#8890 .graf .graf--h3 .graf-after--p name="8890"}

Professionals use knowledge of **public keys**, **TCP connections**, and
